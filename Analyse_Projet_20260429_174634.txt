============================================================
1) Bugs / Risques identifies (priorises)
============================================================

[Critique] Gestion d erreur 401 fragile dans l interceptor
- Fichier: src/app/auth/interceptors/jwt.token.interceptor.ts
- Observations:
  - Utilisation de err.error.message sans garde stricte.
  - Peut casser sur erreurs reseau/proxy si err.error est null/chaine/non objet.
- Risque:
  - Message utilisateur incoherent, erreurs secondaires, comportement de navigation non maitrise.
- Recommandation:
  - Utiliser un extraction safe du message (optional chaining + fallback stable).
  - Ajouter tests unitaires pour cas: 401 JSON, 401 text/plain, status 0, 500.

[Eleve] Regle de route publique trop permissive
- Fichier: src/app/service/data-sharing.service.ts (isPublicRoute)
- Observation:
  - Verification prefixe sur toutes les routes publiques (startsWith) peut marquer des chemins inattendus comme publics (ex: /loginXYZ).
- Risque:
  - Contournement logique auth sur routes mal nommees ou futures routes.
- Recommandation:
  - Garder exact-match pour /login, /inscription.
  - Prefix-match uniquement pour routes tokenisees (ex: /validateEmail/, /resetPassword/).

[Eleve] Service central trop volumineux et couplage fort
- Fichier: src/app/service/data-sharing.service.ts (~1400+ lignes)
- Observation:
  - Service god-object (auth state, notifications, navigation, mails, CRA, cache, etc.).
- Risque:
  - Regressions frequentes, difficultes de test, complexite de maintenance.
- Recommandation:
  - Extraire en facades domaine: AuthStateFacade, NotificationFacade, NavigationFacade, UserProfileFacade.

[Moyen] Bruit de logs massif en production
- Fichiers principaux:
  - src/app/service/data-sharing.service.ts
  - src/app/compo/cra/cra-form/cra-form-cal.component.ts
  - src/app/compo/dashboard/dashboard.component.ts
  - src/app/compo/_utils/mere-component.ts
- Observation:
  - Tres grand nombre de console.log (>= 200 occurrences detectees sur scan partiel).
- Risque:
  - Exposition de donnees, bruit en support, perf degradee.
- Recommandation:
  - Introduire LoggerService avec niveaux + activation par environment.
  - Interdire console.log en prod via lint/CI.

[Moyen] Build/dev scripts fragiles selon shell/environnement
- Fichier: package.json
- Observations:
  - Script build: set NODE_OPTIONS=--openssl-legacy-provider;ng build (syntaxe shell specifique, option obsolete/sensible).
  - Essai local: NODE_OPTIONS flag refuse par Node 14; npx ng build non resolu dans le shell courant.
- Risque:
  - Build non reproductible selon machine/terminal.
- Recommandation:
  - Normaliser scripts cross-platform (cross-env).
  - Fixer version Node/NPM via .nvmrc/.node-version et doc projet.

[Moyen] Incoherences TypeScript remontees
- Fichier: tsconfig.json
- Erreurs/outils:
  - baseUrl deprecie (TS6+)
  - downlevelIteration deprecie
  - moduleResolution=node10 deprecie
  - rootDir implicite problematique
- Recommandation:
  - Planifier upgrade outillage (Angular/TS) + aligner tsconfig.



============================================================
2) Code a ameliorer (qualite, architecture, maintenabilite)
============================================================

2.1 Architecture et separation des responsabilites
- Decouper DataSharingService en modules metier.
- Limiter les side-effects (navigation, storage, appel API) dans un meme flux.
- Introduire interfaces claires pour etats partages (readonly streams + commandes explicites).

2.2 RxJS / abonnements
- Uniformiser pattern de cleanup dans composants: takeUntil(destroy$) ou Subscription composite strict.
- Eviter les subscribe imbriques; preferer switchMap/concatMap selon cas.
- Deja vu en historique projet: risques de boucles login dus a emissions et subscriptions non contraintes.

2.3 Gestion d erreurs
- Centraliser normalisation des erreurs HTTP (status, code metier, message user-friendly).
- Eviter JSON.stringify(error) brut vers UI.
- Ajouter telemetry minimum pour incidents (sans donnees sensibles).

2.4 Securite front
- JWT deja deplace en sessionStorage (bon point recemment).
- Nettoyer toute trace de credentials potentiels dans logs.
- Verifier que chaque route sensible est protegee par guard role-based et pas seulement login.

2.5 Hygiene code
- Remplacer magic strings role/status par enums/constants.
- Renommer methodes ambiguës et harmoniser conventions FR/EN.
- Supprimer code commente mort massif.

2.6 Performance
- Sur ecrans lourds (dashboard/cra-form), limiter recalculs et logs.
- Verifier ChangeDetection (OnPush) + immutabilite des updates listes.
